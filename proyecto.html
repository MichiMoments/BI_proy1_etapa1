def limpieza(data):
    data = data.str.lower()
    data = data.str.replace("", "a").str.replace("", "e").str.replace("", "i").str.replace("", "o").str.replace("", "u")
    data = data.str.replace("[^a-z]", " ")
    return data

data['Review'] = limpieza(data['Review'])

In [ ]:
def benchmark(clf, X_train, y_train, X_test, y_test):
    clf.fit(X_train, y_train)
    pred = clf.predict(X_test)
    cm = confusion_matrix(y_test, pred)
    disp = ConfusionMatrixDisplay(confusion_matrix=cm, display_labels=clf.classes_)
    disp.plot(cmap=plt.cm.Blues)
    
    print("Exactitud: ", round(accuracy_score(y_test, pred), 3))
    print("Precisin: ", round(precision_score(y_test, pred, average='weighted'), 3))
    print("Recall: ", round(recall_score(y_test, pred, average='weighted'), 3))
    print("F1: ", round(f1_score(y_test, pred, average='weighted'), 3))

Resultados¶

Podemos ver que los 3 algoritmos tienen valores similares para sus diferentes mtricas:

Exactitud: Esta mtrica indica que porcentaje de las predicciones que realizo el modelo corresponden a la clase correcta.

Precisin: Esta medida indica que porcentaje de las predicciones son realmente correctas.

Recuperacin: Indica el porcentaje de los casos positivos reales que el modelo detecta.

F1-Score: Es una forma de combinar precisin y recuperacin en una sola medida, por lo que nos resulta til cuando necesitamos un equilibrio entre estas 2 mtricas.

Al analizar los resultados de los algoritmos Ridge Classifier, Logistic Regression y Complement NB, se observa que presentan valores similares en trminos de exactitud, precisin, recuperacin y F1-Score. Esto sugiere que no hay diferencias significativas en el desempeo entre estos tres algoritmos para este problema en especfico.

Sin embargo, encontramos que el modelo que mejor se acerca a lo que buscamos es el Logistic Regression
